# Auth S4 — Owner-Approved Invites + multi-family credential fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Owner-approved family invites (no auto-join) on the S1/S3 backbone, plus the family-agnostic credential fix that lets a user belong to >1 family.

**Architecture:** Migration `0005` (`invites` + `memberships.invite_id`/`created_at`); make `kind='app'` creds family-agnostic (`family_scope=NULL`, membership is the gate) and redefine `/auth/whoami` to memberships; invite endpoints (mint/redeem/approve/decline/revoke/remove/list) via a generalized `ownerGate`, with all multi-statement endpoints on one `pool.connect()` txn and guarded `UPDATE…WHERE status RETURNING` transitions.

**Tech Stack:** TypeScript/Hono on Vercel, `pg` (**pool max:1 on Vercel → txns via `pool.connect()`, never `q()` inside a txn**), vitest vs live Postgres.

**Spec:** `docs/superpowers/specs/2026-06-19-auth-s4-invites-design.md`. **ADR:** 0021 + 0011 §5/§11. Both review passes folded.

## Global Constraints

- API tests from `apps/api`: `DATABASE_URL=postgres:///fad_test npx vitest run` vs live PG. **Test `beforeAll` migration chains MUST load 0001→0005** in order.
- **Vercel pool max:1**: redeem/approve/remove run on ONE `pool.connect()` client, `BEGIN`/`COMMIT`/`ROLLBACK`, all via `client.query`; **never module `q()` inside the txn**. `audit()` (uses `q()`) is called **after COMMIT**. Mirror `device.ts`.
- State transitions use **guarded `UPDATE … WHERE status=$expected RETURNING`** (`rowCount===1`→ok; `0`→re-read to disambiguate), never SELECT-then-UPDATE.
- Owner ops use `ownerGate(c,fid)` = `authorizeTenant` + `role==='owner'` + `cred.kind==='app'` (legacy household `role=null`→403; `kind='cli'`→403). Returns `{sub}` | `{status}`.
- Invite token = `randomBytes(32).base64url` (≥128-bit), SHA-256 at rest, raw returned ONCE, **response not gzipped** (`Cache-Control: no-store, no-transform`). Role allowlist `{adult}`, **never owner**. `max_uses` 1–10. Uniform 404 on bad/expired/exhausted/revoked. Redeem rate-limit+lockout per `account:redeem:<sub>`; success calls `resetFailures`.
- App creds are **family-agnostic** (`family_scope=NULL`); access = per-request `membership(sub, path-fid)`. CLI creds stay family-scoped. Ratelimit helpers: `clientIp`, `hit(key,win,cap)`, `isLocked(key)`, `recordFailure(key,win,threshold,lockSecs)`, `resetFailures(key)`.
- Lazy dynamic `import()` in handlers (match `app.ts`). Regenerate `api/index.js` via `npm run build:fn` at the end. Deferred: QR, deep-link domain, teen, expiry **sweep**, S6 UI, Firebase, E2EE handoff.

```ts
import { randomBytes, createHash } from "node:crypto";
import { q } from "../db.ts";
export const hashInvite = (t: string) => createHash("sha256").update(t, "utf8").digest("hex");
export const genInviteToken = () => randomBytes(32).toString("base64url");
const id = () => "inv_" + randomBytes(9).toString("hex");

export async function createInvite(familyId: string, createdBy: string, mode: "qr"|"link", role: string, maxUses: number) {
  const token = genInviteToken();
  const inviteId = id();
  const ttl = mode === "qr" ? "15 minutes" : "72 hours";
  await q(
    `INSERT INTO invites(id, family_id, role, token_hash, mode, max_uses, created_by, expires_at)
     VALUES ($1,$2,$3,$4,$5,$6,$7, now() + $8::interval)`,
    [inviteId, familyId, role, hashInvite(token), mode, maxUses, createdBy, ttl],
  );
  return { inviteId, token };
}
```

```ts
import { pool } from "../db.ts";
const PENDING_CAP = 20;
export async function redeem(token: string, sub: string) {
  const client = await pool.connect();
  try {
    await client.query("BEGIN");
    const inv = await client.query(
      `SELECT id, family_id, role, used_count, max_uses FROM invites
       WHERE token_hash=$1 AND status='active' AND used_count<max_uses AND expires_at>now() FOR UPDATE`,
      [hashInvite(token)]);
    if (inv.rowCount !== 1) { await client.query("ROLLBACK"); return { notfound: true as const }; }
    const { id: invId, family_id, role } = inv.rows[0];
    const pend = await client.query(`SELECT count(*)::int n FROM memberships WHERE family_id=$1 AND status='pending'`, [family_id]);
    if (pend.rows[0].n >= PENDING_CAP) { await client.query("ROLLBACK"); return { capfull: true as const }; }
    const ins = await client.query(
      `INSERT INTO memberships(user_id, family_id, role, status, invite_id)
       VALUES ($1,$2,$3,'pending',$4) ON CONFLICT (user_id, family_id) DO NOTHING RETURNING 1`,
      [sub, family_id, role, invId]);
    if (ins.rowCount === 1) {
      await client.query(
        `UPDATE invites SET used_count=used_count+1,
           status = CASE WHEN used_count+1 >= max_uses THEN 'exhausted' ELSE 'active' END
         WHERE id=$1 AND status='active'`, [invId]);
      await client.query("COMMIT");
      return { ok: true as const, family_id, role };
    }
    const cur = await client.query(`SELECT status FROM memberships WHERE user_id=$1 AND family_id=$2`, [sub, family_id]);
    await client.query("COMMIT");
    return { conflict: cur.rows[0].status as "pending"|"active"|"removed" };
  } catch (e) { await client.query("ROLLBACK"); throw e; } finally { client.release(); }
}
```

app.post("/invites:redeem", async (c) => {
  const t = bearer(c); if (!t) return c.body(null, 401);

  const key = `account:redeem:${sub}`;
  if (await isLocked(key)) return c.body(null, 429);
  const body = await c.req.json().catch(() => null);
  if (!body?.token) return c.json({ type:"bad-request" }, 400);
  const { redeem } = await import("./auth/invites.ts");
  const out = await redeem(body.token, sub);
  const { audit } = await import("./auth/audit.ts");
  if ("notfound" in out) { await recordFailure(key, 900, 5, 900); return c.body(null, 404); }

  await resetFailures(key);                                               // [M1]
  if ("conflict" in out) {

    return c.json({ type: out.conflict === "active" ? "already-member" : "removed" }, 409);

  const fam = await q(`SELECT name FROM families WHERE id=$1`, [out.family_id]);
  await audit("invite.redeem", { actorUserId: sub, familyId: out.family_id });
  return c.json({ family_id: out.family_id, family_name: fam.rows[0]?.name, role: out.role, status: "pending" }, 200);
});
