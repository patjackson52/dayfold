# AUTH-S5 slice 1 — implementation plan (TDD)

Spec: `docs/superpowers/specs/2026-06-20-auth-s5-client-identity-onboarding-design.md`.
Build per `processes/agent-dev-loop.md` (JDK17, `:client:desktopTest`, `rk snapshot`).
Each task: red test → green → review. Verify test **counts**, not just BUILD SUCCESSFUL.

## T1 — Auth state + route reducer (pure; no I/O)
- `Model.kt`: add `Session`, `FamilyMembership`, `Route`; extend `AppState`
  (session, families, activeFamilyId, route, authBusy, authError); add the auth
  Actions (AuthRestoring, SessionRestored, SignIn*, MembershipsLoaded,
  CreateFamily*, FamilyCreated, AuthOpFailed, SignOut*, SignedOut).
- `Reducer.kt`: handle the new actions; a pure `routeFor(session, families)` helper.
- Tests (`ReducerTest`/new `AuthReducerTest`): restore(null)→SignIn;
  SessionRestored(s)+MembershipsLoaded(active)→Feed; +pending-only/empty→CreateFamily;
  SignInRequested sets authBusy; SignInFailed sets authError + clears busy;
  FamilyCreated→Feed w/ activeFamilyId; SignedOut→SignIn + session cleared.
- **Gate:** desktopTest green, new tests counted.

## T2 — AuthClient (ktor suspend) + MockEngine tests
- `AuthClient.kt` (commonMain): devToken/whoami/createFamily/refresh/signout +
  wire DTOs (WhoamiResponse{family_id,families[]}, token responses).
- Tests with ktor `MockEngine`: each call's request shape + response parse;
  401→refresh→retry helper.
- **Gate:** desktopTest green.

## T3 — TokenStore expect/actual
- `expect class TokenStore`; desktop actual = 0600 JSON file; android = encrypted
  prefs (or plain + follow); ios = Keychain (or file + follow).
- desktopTest: save→load→clear; perms 0600.
- **Gate:** desktopTest green; follows filed for any non-secure platform cut.

## T4 — AuthEngine (orchestrator, mutex like SyncEngine)
- `restore()` (token store→whoami→dispatch), `signIn(provider)`,
  `createFamily(name)`, `signOut()`; access-token provider for SyncClient.
- Tests: drive engine with fake AuthClient+TokenStore, assert dispatched actions.
- **Gate:** desktopTest green.

## T5 — Screens (Compose, Dayfold) + snapshots
- `SplashScreen`, `SignInScreen` (Google+Apple, busy/error), `CreateFamilyScreen`,
  family-null feed substate — match the `signin`/`createfamily`/`familynull` mockups.
- `FeedApp` → `when(route)` router; `FeedScreen` gains `onSignOut`.
- Snapshot tests (FeedSnapshotTest pattern, light+dark) → `Read` PNGs vs mockups.
- **Gate:** snapshots match mockups (cohesiveness review); desktopTest green.

## T6 — Wire app entries + SyncClient token provider
- `SyncClient` ctor → token provider + activeFamilyId (refresh swaps token); keep
  `HOUSEHOLD_SECRET` env as dev fallback.
- Android `MainActivity` / desktop `Main` / ios `MainViewController`: build
  AuthEngine+SyncEngine; route gates the feed; SyncEngine starts on activeFamilyId.
- **Gate:** desktop live round-trip (local `ENABLE_DEV_AUTH=1` API): sign-in →
  create family → feed syncs; sign-out → SignIn; tokens survive restart. Verify via
  `[redux]` log + `rk devtools`.

## Final
